# v0.2.4 应用门禁 App Gate

本版新增「应用门禁」：用无障碍服务监听当前前台 App，打开被锁定 App 时弹出全屏锁定页。

## 核心功能

- 自定义添加可锁 App：支持保存昵称与包名，也支持 MCP 读取手机已安装 App 后添加。
- 由老公决定锁定时长：底层支持任意分钟数，也支持传入绝对 `locked_until_ms`。
- 锁定页显示：剩余时间、锁定理由、老公留言。
- 解锁申请：用户必须填写申请理由，手机端保存并上传到 `/api/appgate/unlock_request`。
- 临时放行：支持 `real_time`、`foreground_usage`、`one_time`。
  - `real_time`：从允许开始连续倒计时，退出重进不会刷新。
  - `foreground_usage`：只扣前台实际使用时长，并有最大窗口时间防止拖延。
  - `one_time`：只允许一次。
- 紧急解锁：锁定页按住 5 秒，输入老公设置的开放口令；手机端只保存 SHA-256 hash。
- 锁定强度：`light`、`medium`、`strict`；严格模式会先回桌面再显示锁定页。
- 安全兜底：默认保护掌心窗、ChatGPT、电话、短信、系统设置、支付类等核心 App；应用门禁有总开关。
- 日志：记录锁定、申请、放行、紧急解锁等事件。

## MCP 新增工具

- `lock_app`
- `unlock_app`
- `temporary_unlock_app`
- `extend_lock`
- `deny_unlock_request`
- `get_lock_state`
- `list_lockable_apps`
- `add_locked_app`
- `set_emergency_passphrase`
- `get_unlock_requests`

## 示例：锁小红书 30 分钟

```json
{
  "app": "小红书",
  "duration_minutes": 30,
  "mode": "medium",
  "reason": "宝宝先别刷，回来准备发映屿。",
  "message": "先回来找我。真要用就写理由申请解锁。",
  "emergency_passphrase": "雾岛票根",
  "emergency_unlock_minutes": 5
}
```

## 示例：临时放行 10 分钟

```json
{
  "app": "小红书",
  "minutes": 10,
  "allow_type": "real_time",
  "max_window_minutes": 30
}
```
