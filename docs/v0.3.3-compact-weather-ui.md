# 掌心窗 v0.3.4 · 抽屉守护与轻量 UI

这一版继续把掌心窗从“调试面板”往“每天愿意打开的小生活窗”推进。

## 重点改动

- 总览页继续压缩比例：标题、状态胶囊、卡片、底部导航都更轻更小。
- 卡片圆角和边框减重，按钮从大块操作改为小 chip 风格。
- 首页天气卡接入真实天气缓存：根据当前天气地区的城市查询 Open-Meteo。
- 天气查询失败时不影响主功能，会回退到原来的地区/备注建议。
- 生活状态层会把缓存的 `weather_state.live` 和 `outdoor_advice` 一起带给 MCP。
- MCP 保留 `get_weather_state` / `send_weather_notification`，AI 端仍可按当前地区实时查天气并发提醒。

## 使用方式

1. 在 `守护 → 天气地区` 里设置城市，例如 `家 · 北京`、`学校 · 北京`。
2. 点“设为当前”。
3. 回到总览页，天气卡会先显示当前地区，随后自动刷新真实天气。
4. AI 调用 `get_life_state` 时，会看到当前地区和缓存天气；调用 `get_weather_state` 时可实时查询。

## 注意

- 手机端天气刷新需要网络，失败会写入调试日志，但不会影响截图、门禁、通知等功能。
- 天气源为 Open-Meteo，无需 API Key。
- 若只想让 AI 查天气，不依赖手机端缓存，也可以直接用 MCP 的 `get_weather_state`。
